# 伊都文化会館 大ホール 空席情報システム セットアップ手順

---

## ファイル一覧

| ファイル名 | 役割 |
|---|---|
| `gas_api.gs` | Google Apps Script コード（GASエディタに貼り付ける） |
| `index.html` | 公開用の座席表ページ（ブラウザで開くだけで動作） |
| `README.md` | このセットアップ手順書 |

---

## Step 1：Googleスプレッドシートを作成する

1. [Google ドライブ](https://drive.google.com) を開き、**「新規」→「Googleスプレッドシート」** を作成します。
2. シート名（画面下部のタブ）を **`seats`** に変更します。
3. A1セルに以下のヘッダーを手入力します（後述の `initSeats()` を実行すれば自動入力されるため省略可）。

   | A列 | B列 | C列 |
   |---|---|---|
   | row | seat_number | reserved |

---

## Step 2：Google Apps Script をセットアップする

1. スプレッドシートのメニューから **「拡張機能」→「Apps Script」** を開きます。
2. デフォルトで表示されている `コード.gs` の内容を**すべて削除**します。
3. `gas_api.gs` の内容を**コピー＆ペースト**します。
4. 上部の **「保存」ボタン（フロッピーディスクアイコン）** をクリックします。

---

## Step 3：初期座席データを一括生成する

1. GASエディタの上部にある関数選択プルダウンから **`initSeats`** を選択します。
2. **「実行」ボタン（▶）** をクリックします。
3. 初回実行時はGoogleアカウントへのアクセス許可を求められます。**「権限を確認」→「許可」** をクリックします。
4. スプレッドシートに戻り、`seats` シートに座席データ（全座席 `FALSE`）が生成されていれば成功です。

> **生成される座席数の内訳**
> - A〜G列（7列）：各44席（左8 + 中央24 + 右12）= 308席
> - H〜O列（8列）：各46席（左10 + 中央24 + 右12）= 368席
> - **合計：676席**

---

## Step 4：GAS Web アプリとしてデプロイする

1. GASエディタ右上の **「デプロイ」→「新しいデプロイ」** をクリックします。
2. 「種類の選択」で **「ウェブアプリ」** を選択します。
3. 以下の設定を行います：

   | 設定項目 | 値 |
   |---|---|
   | 説明 | （任意：例「空席API」） |
   | 次のユーザーとして実行 | 自分（自分のGoogleアカウント） |
   | アクセスできるユーザー | **全員** |

4. **「デプロイ」** をクリックします。
5. 再度アクセス許可を求められる場合は **「アクセスを承認」** をクリックします。
6. デプロイ完了後に表示される **「ウェブアプリのURL」** をコピーして控えておきます。

   ```
   例：https://script.google.com/macros/s/XXXXXXXXXXXX/exec
   ```

---

## Step 5：index.html に API URL を設定する

1. `index.html` をテキストエディタで開きます。
2. 以下の行を探します：

   ```javascript
   const GAS_API_URL = "ここにGAS WebアプリのURLを貼る";
   ```

3. 引用符内をStep 4でコピーしたURLに書き換えます：

   ```javascript
   const GAS_API_URL = "https://script.google.com/macros/s/XXXXXXXXXXXX/exec";
   ```

4. ファイルを保存します。

---

## Step 6：動作確認

### API の確認
1. ブラウザのアドレスバーに Step 4 の URL を貼り付けてアクセスします。
2. 以下のようなJSONが表示されれば正常です：

   ```json
   {
     "seats": [
       { "row": "A", "seat_number": 3, "reserved": false },
       ...
     ],
     "updated_at": "2025-03-16T10:00:00+09:00"
   }
   ```

### 座席表ページの確認
1. `index.html` をブラウザで開きます（ダブルクリックまたはドラッグ＆ドロップ）。
2. 座席表が表示され、5秒ごとに自動更新されることを確認します。

### 予約済フラグのテスト
1. スプレッドシートの `seats` シートで、任意の行の C列 を `TRUE` に変更します。
2. `index.html` で該当座席がグレーに変わることを確認します（最大5秒後に反映）。

---

## 座席の予約管理方法

予約状況の更新は、スプレッドシートの **C列（reserved）** を直接編集します。

| 値 | 意味 |
|---|---|
| `FALSE` | 空席 |
| `TRUE` | 予約済 |

---

## トラブルシューティング

| 症状 | 対処法 |
|---|---|
| 「データ取得に失敗しました」が表示される | GAS_API_URL が正しいか確認。デプロイ設定「アクセスできるユーザー：全員」になっているか確認 |
| 座席が表示されない | ブラウザのコンソールでエラーを確認。URLの更新が必要な場合は「新しいデプロイ」を再作成 |
| スプレッドシートを更新しても反映されない | GASは「デプロイ済みバージョン」を参照するため、コードを変更した場合は再デプロイが必要（データ変更は不要） |
| ローカルで `index.html` を開くとCORSエラーが出る | GAS側のCORSヘッダーは `ContentService` が自動付与します。エラーが続く場合はWebサーバー経由で配信してください |
